# NRIN — CLINICAL INTELLIGENCE V1

## Purpose

Define the future clinical intelligence layer for NRIN.

This layer is not a standardized treatment-plan engine.

It is:
**a structured way to capture what practitioners have found helpful in real case patterns**

The goal is to preserve and surface:
- practical wisdom
- working approaches
- useful assignments
- contextual counselor insight

without forcing rigid templates or generic treatment plans.

---

## Core Principle

NRIN should not tell counselors how to treat.

NRIN should help capture:
> **what has actually helped in similar situations**

This is a learning layer, not a command layer.

---

## What This Is

A future system for collecting and surfacing:

- case-pattern-based counselor insights
- optional worksheets / prompts / assignments
- practical notes on what helped
- optional notes on what did not work as expected
- patterns that may later correlate with better outcomes

---

## What This Is NOT

Do NOT turn this into:

- a static resource library
- a generic worksheet dump
- standardized treatment plans
- a compliance-heavy documentation burden
- a polished educational portal too early

This should remain:
- practical
- lightweight
- contextual
- experience-backed

---

## Key Philosophy

Counselors do not think:
> “I want to browse worksheets.”

They think:
> “I have a patient like this — what has actually helped?”

NRIN should eventually answer that question.

---

## Contribution Model (Future)

The system should eventually allow counselors to contribute in a lightweight way.

### Contribution shape

#### 1. Situation / case pattern
Examples:
- early recovery
- relapse cycle
- trauma history
- anger issues
- family estrangement
- court pressure
- domestic violence history
- eating disorder crossover
- grief
- shame / resentment
- co-occurring mental health concerns

---

#### 2. What helped
Short practical note describing:
- approach
- framing
- exercise
- intervention
- sequencing
- counselor strategy

---

#### 3. Optional artifact
Examples:
- worksheet
- writing prompt
- reflection prompt
- assignment
- group exercise
- treatment-planning aid

---

#### 4. Optional note on what did not work
Not:
“failure reporting”

But:
“what did not work as expected”

This should remain optional to reduce defensiveness and increase honesty.

---

## Reward Philosophy

The system should not reward:
- volume
- polish
- marketing language
- self-promotion

The system should eventually reward:
- usefulness
- reuse
- trust
- contribution quality over time

Core rule:
> **reward usefulness, not volume**

---

## Why This Matters

This layer could eventually become highly valuable because it captures:

- tacit counselor knowledge
- practical field-tested approaches
- generational knowledge transfer
- patterns not visible in formal literature
- strategies that improve engagement and retention

---

## Important Constraint

This should not be built now.

Reason:
- current mission remains patient matching, brochure surface, activation flow, and facility coordination
- premature expansion would create drift
- this layer will be much more valuable once real cases are flowing through NRIN

---

## What To Scaffold Now

Only:
- this brain/spec document
- future conceptual data shape
- awareness that the system should support this later

Do NOT build now:
- UI
- upload workflows
- ranking system
- contributor profiles
- reputation system
- moderation engine
- artifact repository
- search interface
