# NRIN DESIGN DOCTRINE — REFINEMENT

## CORE PRINCIPLE

Refinement is not a filter panel.
It is a guided decision conversation.

---

## COLOR SYSTEM (LOCK)

- BLUE → patient choice
- INDIGO → facility identity
- STONE → negative / muted / caution

Never mix these roles.

---

## INTERACTION RULES

Selections must be:
- explicit (checkmark or clear state)
- additive (not subtractive confusion)
- reversible

---

## LANGUAGE RULE

Do NOT imply failure.

Avoid:
- "something feels off"

Use:
- "let’s narrow this down"
- "let’s find the best fit"

---

## UI STRUCTURE RULE

Prefer:
- progressive questions
- one decision at a time

Avoid:
- dense control panels

---

## FUTURE DIRECTION

Refinement → conversational UI
not → dashboard
